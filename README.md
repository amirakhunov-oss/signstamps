# SignStamps

SignStamps - веб-приложение для поиска и проверки подписей и печатей на сканах PDF-документов.

Проект объединяет FastAPI backend, интерфейс ручной проверки, YOLO-детекцию объектов, сопоставление найденных подписей и печатей с эталонами, а также human-in-the-loop процесс для исправления ошибок модели и накопления новых обучающих примеров.

Сценарий использования: сотрудник загружает PDF-документ, система преобразует страницы в изображения, находит подписи и печати, показывает результат проверки и позволяет вручную подтвердить или исправить спорные случаи.

## Возможности

- Загрузка PDF-документов с фоновой обработкой.
- Конвертация страниц PDF в изображения.
- YOLO-инференс для классов `signature` и `stamp`.
- Вырезание crop-изображений найденных объектов.
- Сводка статусов по документу и отдельным страницам.
- База эталонных подписей и печатей.
- Указание ожидаемых организаций и подписантов для документа.
- Визуальная проверка страниц с bounding boxes.
- Карточки найденных объектов с уверенностью модели и статусом сопоставления.
- Ручное подтверждение проверенного документа.
- Ручное добавление пропущенной подписи или печати.
- Экспорт ручных исправлений в формат для последующего дообучения модели.

## Архитектура

```text
Загрузка PDF
  -> рендеринг страниц
  -> YOLO-детекция подписей и печатей
  -> вырезание найденных областей
  -> сопоставление с эталонами
  -> расчет статуса документа и страниц
  -> ручная проверка и корректировка
  -> сохранение примеров для active learning
```

Backend хранит документы, страницы, детекции, эталоны и ручные корректировки в SQLite. Изображения страниц и crop-области раздаются через FastAPI, поэтому проверяющий может смотреть результат прямо в браузере.

## Computer Vision Pipeline

Детектор работает с двумя классами:

- `signature`
- `stamp`

Приложение запускает YOLO-инференс по отрендеренным страницам документа и сохраняет для каждой детекции координаты, confidence, источник, путь к crop-изображению, score сопоставления и итоговый статус.

Сопоставление с эталонами реализовано через OpenCV. Pipeline нормализует изображения и сравнивает их по нескольким признакам: similarity масок, template correlation, perceptual hash, Chamfer similarity и ORB features. Такой подход позволяет получить рабочий прототип без отдельной embedding-модели.

## Процесс проверки

Пользователь может:

- загрузить один или несколько PDF-файлов;
- указать ожидаемые организации и подписи;
- посмотреть результат проверки по каждой странице;
- назначить найденный объект нужному эталону;
- вручную добавить пропущенную подпись или печать;
- подтвердить документ после проверки.

Ручные исправления сохраняются в YOLO-совместимом формате. Поэтому приложение можно использовать не только как inference demo, но и как инструмент улучшения датасета.

  app.py                  FastAPI-приложение и CV pipeline

## Локальный запуск

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r accountant_tool\requirements.txt
.\.venv\Scripts\python.exe -m uvicorn accountant_tool.app:app --host 127.0.0.1 --port 8011
```

Укажите путь к YOLO-весам через переменную `SIGNSTAMP_YOLO_MODEL`:

```powershell
$env:SIGNSTAMP_YOLO_MODEL = "C:\path\to\best.pt"
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8011
```
